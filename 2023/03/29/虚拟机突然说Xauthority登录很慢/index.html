Liunx创建新用户登录异常：/usr/bin/xauth: error/timeout in locking authority file /home/liuqidong/.Xauthority

问题1：在服务器上创建新的用户userA，在使用ssh连接时报错：/usr/bin/xauth: error in locking authority file /home/userA/.Xauthority
解决方案：报错的意思就是找不到/home/userA/.Xauthority。这个文件按理来说是登录用户时会自动创建的，为什么会找不到呢？发现是应为/home/下没有创建/userA/文件夹。因此，输入命令后重新连接：

	sudo mkdir /home/userA

  • 1

问题2：在重启后，发现ssh连接特别慢。不仅如此，连接上又报错： /usr/bin/xauth: timeout in locking authority file /home/userA/.Xauthority
解决方案：看问题也就知道为什么连接很慢了，因为timeout了嘛。为什么会timeout呢？因为没有给userA对于/home/userA/的写权限，所以导致在连接时无法创建文件.Xauthority。因此，输入下面命令后重启：

	chown userA:userA -R /home/userA

  • 1

问题3：没想到重启后还是有问题！报错：/usr/bin/xauth: file /home/spring/.Xauthority does not exist。并且命令行一直有问题：输入Backspace，命令行中不是退格，而是^H。
解决方案：从命令行输入问题可以想到，应该是创建用户时，没有个用户bash，因此，输入下面命令后重启：

	usermod -s /bin/bash userA

  • 1

终于，解决完上述问题后可以正常用ssh连接新创建的用户了！
